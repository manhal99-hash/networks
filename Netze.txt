Wellnwiederstand
- Charakterstische Größe eines Kabels, die durch Geometrie und Meterial des Kbelaufbaus bestimmt wird. ist kein Qualitätsmerkmal, sollte aber überall gleich sein
- Änderungen des Wellenwiederstands führen zu Reflexionen des Signals

Aufgabe : was passiert wenn versehentlich 2 Kabel mit verschiedenen Wellenwiederstande aneinenader angeschlossen werden ? 
Es gibt Reflexionen, ein Teil des Signals wird zum Sender reflexiert

NEXT : near end cross talk „Übersprechen“

ACR : attenuation to cross talk ratio

ACR = NEXT - a
a = Dämpfung (Signalverlust)

- Im ACR werden im Grunde das Überprechen (NEXT) und die Dämpfung zusammengefasst
- Der ACR Wert sollte bei einem Kabel möglichst groß sein, beschreibt die Qualität eines Kabels

Lokale Netze, Ethernet und WLAN




LAN : Local Area Network 
1- Ethernet : ist eine Kabelgebundene Übertragung  von Daten in lokalen Netze, mit dezentralem Medienzugriff und verschiedene Kabelvarianten
2- WLAN : Wirless Local Area Network, ist ein drahtlose Übertragung von Daten in lokalen Netze, mit dezetralen stochastischen Medienzugriff und verschiedene Übertragungsarten.

MAC : Media Access Control (Hardware Adresse)


Medien-Zugriffsprotokoll bei Ethernt u. WLAN :

- CSMA / CD : Carrier Sense Multiple Access / Collision Detection : ist Zugriffsverfahren bei Ethernet basiert auf dem gleichen Ansatz.
Eigenschaften : nicht deterministich, keine garantierte Antwortzeit

Wie funktioniert das CSMA / CD bei Ethernet  ?
- Ethernet verwendet als Zugriffsverfahren CSMA / CD
- Jede Station überwacht das Medium (carrier sensing, listen before talking) problem ist : „jeder hört mit“
- bei freiem Medium sendet eine Station nach einer kurzen Wartezeit
- Falls das Medium belegt ist, wartet die Station bis das Medium frei ist und beginnt nach einer zufälligen Zeit mit dem Senden
- es kann passieren dass mehere Stationen gleichzeitig senden, dann kommt zu einer Kollision
- daher wird das Medium auch während der Sendung weiter abgehört
- ein Kollision wird vom Sender dadurch entdeckt, bei einer Kollision wird die Datenübertragung sofort abgebrochen und es wird ein spezielles Störsignal gesendet
- nach der Sendung der Störsignal wird eine bestimmte Zeit + Zufallwartezeit gewartet und wieder gesendet

-CSMA / CA : Carrier Sense Maultiple Access / Collision Avoidence : ist Zugriffsverfahren bei WLAN.

wie funktioniert das CSMA / CA ?
-Die Sendstation hört das Medium ab
- ist das Medium frei? darf sie nach DIFS plus zufälliger Wartezeit senden
- ist das Medium nicht frei ? wartet sie bis das Medium frei wird und sendet dann nach DIFS plus zufälliger wartezeit 
- Der Empfänger bestätigt das Erhalten mit einem ACK. Nach erhalten der ACK ist die Datenübertragung abgeschlossen.

wichtige Frage : wann treten Kollisionen auf ?
- wenn zwei Stationen gleichzeitig zu senden beginnen, wenn sie z.b zufälligen wartezeit erhalten
- danach wird die zufällige wartezeit verdopplet

TCP : bietet eine zuverlässige, verbindungsorientierte Service zur Übertragung von Byte-Strom unter Beachtung der aktuellen Lastsituation


Automatic Repeat Request (ARQ) :
strategie zur Erkennung und Korrektur des Verlusts ganzer Nachrichten 

1-  SENDER fordert vom Empfänger eine Bestätigung (ACK) dass die Daten korrekt empfangen wurden
2- kommt nach Ablauf eines Timeouts keine Bestätigung, wird das Paket erneut gesendet 

Stop and Wait : sender schickt ein Paket zu Empfänger, und wartet bis eine ACK kommt, Sender wartet eine bestimmte Zeit bis er nochmal schickt.
// aber das ist uneiffizienz

Sliding Window : der Sender darf nicht nur ein Segment versenden sondern N, bevor er die ACK erwartet, Segmente werden nummieriert und entsteht ein WINDOW, solange ein Segment bestätigt wird verschiebt sich Anfang und Ende des Window.

Wie geht man mit Übertragungsfehlern um ?

GO-BACK-N :
Empfänger : wenn ein Segment mit Sequenznummer n empfangen wird und die Reihenfolge korrekt ist, sende ein ACK(n)
sollte eine flasche Sequenznummer eintreffen (n+2), sende weiterhin ACKSs mit n bis n+1 eintrifft

Selective Repeat : 
Empfänger : individuelle Bestätigung der Segmente, selbst wenn diese in falscher Reihenfolge eintreffen
Empfangspuffer , um auf fehlende Lücken warten zu können.



Selective Repeat / Go back N erläutern : 
Selective Repeat:
Es wird nur das fehlende oder fehlerhafte Paket erneut gesendet. Alle
nachfolgenden, bereits gesendeten werden beim Empfänger
zwischengespeichert.
Go back N:
Ab dem fehlenden oder fehlerhaften Paket werden nochmals alle Pakete
gesendet. Nach einem fehlerhaft empfangenen werden alle Pakete verworfen
und nicht zwischengespeichert

Welches ARQ Verfahren ist am effizientesten?
Selective Repeat: Es werden nur fehlerhafte Pakete wiederholt



Kapitel 9 TCP Congestion / Flow Control , Quic



Flow Control : Sender schickt zu viele schnelle Daten für Empfänger
Lösung : Empfänger teilt dem Sender den freien Speicher im Buffer mit.

Congestion Control : viele Sender schicken zu viele schnelle Daten für Empfänger und überlasten das Netzwerk 
Lösung :  AIMD (Additive Increase Multiplicative Decrease) : Sender erhöhen die Paketrate sukzessive und verringern diese im Fall von Paketverlusten
1- Additive Increase : erhöhen die Anzahl an zu übertragenen Daten
2- Multiplicative Decrease : habliere /2 die Anzahl von übertragene Daten im Fall von Verlusten

TCP Slow Start : sender startet mit dem Transfer des ersten Datensegments und wartet dann auf die zugehörige Quittung. dann werden verdoppelt wenn Empfänger ACK schickt.


QUIC : bietet die gleichen Sevices wie TCP (zuverlässig, flow Control, Congestion Control)
ABER : 
1- QUIC läuft inder Anwendung (user-space) nicht im Betriebsystem (kern-space)
2- QUIC nutzt UDP zur Übertragung.
Kapitel 10 Angriff auf TCP / UDP


Sicherheitsziele :
1- Inegrität : die Echtheit und genauigkeit von Daten 
2- Vertraulichkeit : gewährleistet dass nur befugte Personen die Nachricht lesen können
3- Verfügbarkeit : gewährleistet dass Benututzer jederzeit oder auf Anfrage Informationen bereitegestellt werden

Computer-Netzwerke Angriff :
1- Spoofing : Angreifer gibt sich als gefälschte Identiät aus um Vorteile ausnutzen
2- MIM : Man in the Middle : Angreifer hört den Datenverkehr / Kanal ab und nutzt die aus 
3- DOS / DDOS : Denial of Serive / Distributed Denial of Service : zu viele Anfragen auf Ziel schiken um den zu überflotten 

a) Dos Angriff auf Bandbreite : mehr Datenverkehr auf dem Router erzeugen mehr als dieser verkraften kann 
b) Dos auf Ressourcen : Mehr Daten auf bestimmte Software oder das gesmate Betribsystem eines Hosts zum Absturz bringen 


DDOS Angriffe via UDP :
1- Reflection Angriffe : 
- Der Angreifer sendet ein Paket via IP/UDP mit der Adresse des Opfers als gefälschte Abesender-IP-Adresse an einen Dienst der auf einem Netzwerk server läuft.
- Der Server antwortet auf die Anfrage und sendet diese ans Opfer weiter 
2- Amplification Angriff : 
- eine besondere Art von Reflection
- eine kleine Anfrage (Payload) des Angreifers führt zu einer großen Antweort des Reflectors

DDOS Angriffe via TCP : 

SYN-Flood : über die 3 way handshake :
Angreifer schickt eine Anfrage an Server (SYN)
Server antwortet mit einem SYN-ACK und wartet auf die Bestätigung (ACK)
Angreifer schickt keine ACK, sondern viele weitere SYN, so wird die SYN-Queue überflottet 

RST(Reset) : Angreifer schickt mit geeälschter Daten und gefälschte Paket (sequenznummer, portnummer, Adresse) gefälschte RST an Server.
so wird die Verbindung zwischen Opfer und Server abgebrochen

löst IP-Adressen in Namen auf „Menschen merken sich keine Nummer sondern Namen“

Domain Name Struktur :
1- TLD : Toplevel Domain : com , de
2- Second Domain :  facebook
3- Subdomain : www

Typen von DNS Server 
1- autorativer Namenserver : gibt gesicherte Antworten für Domainanfragen einer Zone (root Namenserver, TLD, primary und secondary)
2- nicht-autorativer Namenserver : resursive resolver, DNS Informatinen aus zweiter oder dritter Hand.
